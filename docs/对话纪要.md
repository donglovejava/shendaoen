## 第 6 次对话 — 第 2 步：用户注册登录 + Session 鉴权 · 2026-04-21

### 1. 你的话

> 已经成功运行，密码已经修改，每次对话写入 docs/对话纪要.md，请进行下一步。

### 2. 我的思考过程

- 骨架已验证通过，直接进入业务第一块：**鉴权**。这是后面所有功能的入口。
- **不用 Spring Security 默认登录页**（样式丑，无法品牌化）——自己写 Thymeleaf 登录/注册页。
- **从数据库读用户**：实现 `UserDetailsService`，替代启动日志里那个 `inMemoryUserDetailsManager`。之后启动日志里那行 "Using generated security password" 会消失。
- **角色权限分层设计**：
  - `/admin/**` → 仅 ADMIN
  - `/teacher/**`、`/api/teacher/**` → TEACHER 或 ADMIN
  - 其它受保护路径 → 登录即可
  - `/`、`/login`、`/register`、`/ping`、静态资源 → 放行
- **API 与页面分流**：`/api/**` 未登录返回 `401 JSON`（不跳转登录页，便于前端 AJAX）；其它受保护页面会自动跳 `/login`。
- **注册只允许学生角色**：教师/管理员由数据库预置或后续由管理员创建，避免任何人注册成教师刷数据。
- **UI 风格**：渐变紫色背景 + 白色卡片，简洁现代，一套 CSS 贯穿全站。
- 用到 `sec:authorize` 按角色显示不同功能块 → 需要加 `thymeleaf-extras-springsecurity6`。

### 3. 处理过程

**新增文件**：

- `service/CustomUserDetailsService.java` — 从 `user` 表加载用户，角色映射为 `ROLE_xxx`
- `service/UserService.java` — 注册逻辑（用户名去重 + BCrypt 加密 + 固定 STUDENT 角色）
- `controller/AuthController.java` — `/login`（GET 页面）、`/register`（GET + POST）
- `controller/HomeController.java` — 改造：`/` 重定向到 `/home`，`/home` 渲染个人页；`/ping` 保留
- `templates/login.html` — 登录页，支持错误/注册成功/登出提示
- `templates/register.html` — 注册页
- `templates/home.html` — 登录后首页，按角色显示不同功能区块
- `static/css/style.css` — 通用样式（渐变背景、卡片、表单、导航栏）

**修改文件**：

- `pom.xml` — 加 `spring-boot-starter-thymeleaf` 和 `thymeleaf-extras-springsecurity6`
- `config/SecurityConfig.java` — 完整配置：formLogin、角色授权、`/api/**` 走 401、session 单点登录、静态资源放行

### 4. 结果

## 第 7 次对话 — 修复登录密码 + 第 3 步：课程与文件上传 · 2026-04-21

### 1. 你的话

> 用户名/密码/角色（三个 123456 测试账号）登不进去，修好后进行下一步。

### 2. 我的思考过程

**关于登录失败**：
- 根因：`data.sql` 里的 BCrypt 哈希不是我本地实时加密生成的，可能因为复制/编码问题与 Spring Security 的 `matches()` 不兼容。
- 最稳方案：不在 SQL 里写死哈希，改为"启动时用代码里的 `PasswordEncoder` 重新加密 `123456` 写入数据库"——这样哈希必然与验证逻辑一致。
- 用 `CommandLineRunner` 实现，启动时检测到测试账号密码为空或 `matches()` 失败就覆盖。既能修你现在的库，也保证以后任何环境启动后都能用。

**关于第 3 步**：
- 学生浏览 vs 教师管理分成两个 controller，路径前缀 `/courses` 和 `/teacher/courses`，配合 Security 的 `/teacher/**` 角色限制。
- 文件物理存储：每个课程一个子目录 `uploads/course-{id}/`，文件名用 UUID 避免冲突，原始扩展名保留。
- 前端访问：注册 `ResourceHandler` 把 `/files/**` 映射到 `uploads/` 绝对路径，视频用 `<video>` 标签 + `src="/files/xxx"`，PPT 直接下载链接。
- 一次性交付，因为这一步的 5 个页面 + 2 个 controller + 3 个 service 都紧密耦合，分拆反而不好测。

### 3. 处理过程

**修复登录**：
- 改 `src/main/resources/db/data.sql`：密码字段清空
- 新增 `config/DataInitializer.java`：`CommandLineRunner` 启动时检测并重写 3 个测试账号的密码

**第 3 步新增**：
- `config/WebMvcConfig.java` — 映射 `/files/**` → `uploads/` 目录
- `service/FileStorageService.java` — UUID 文件名 + 按课程分子目录
- `service/CourseService.java` — 课程与章节的增删改查
- `controller/CourseController.java` — 学生侧：`/courses` `/courses/{id}` `/chapters/{id}`
- `controller/TeacherController.java` — 教师侧：创建/删除课程、管理章节、上传视频 PPT
- `resources/mapper/CourseMapper.xml` + `ChapterMapper.xml`
